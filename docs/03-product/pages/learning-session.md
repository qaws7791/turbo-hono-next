# 풀스크린 학습 세션 모드 (/session)

## 개요

학습은 홈에서 시작되지만, 실제 학습은 **몰입형 풀스크린 세션 화면**에서 진행됩니다. 별도 '페이지'로 늘리지 않고 "모드(View)"로 제공하여 구조는 단순하게 유지합니다.

---

## 의사결정 근거

- 몰입을 위해 주변 요소를 제거해야 함 (Cognitive Load Theory)
- 학습은 '흐름'이므로 단계형 인터랙션 필요
- 진단을 세션에 녹여 넣어 심리적 부담을 낮춤

---

## 세션 단계 구성 (예: 20분)

| 단계 | 이름                  | 설명                                    |
| ---- | --------------------- | --------------------------------------- |
| 1    | **신규 개념 학습**    | 최적 분량의 새로운 콘텐츠 제공          |
| 2    | **이해도 체크**       | 시험이 아닌 자연스러운 대화/선택형 확인 |
| 3    | **적용 활동**         | 짧은 실전 적용 (Deliberate Practice)    |
| 4    | **Concept 자동 저장** | 새 개념/오답/노트 자동 정리             |
| 5    | **세션 요약 생성**    | 홈에 요약 카드로 표시                   |

---

## 세션 내 이해도 체크 (저마찰 신호)

세션 안에서 자연스럽게 학습 상태를 파악합니다.

### 신호 유형

| 유형           | 설명                                    |
| -------------- | --------------------------------------- |
| 확인 질문      | 자연스러운 질문 (Implicit/Low-friction) |
| 자기평가       | 클릭 1~3회로 간단히 응답                |
| 학습 행동 신호 | 반응, 오답, 설명 요청 등                |

> **참고**: 이해도 체크 결과는 Plan 자동 변경에 직접 연결하지 않음
> 대신 사용자에게 보충 학습 제안이나 학습 리소스 추천 제공 가능

---

## UI 원칙

### 단일 집중

- 한 화면에 한 가지 핵심 행동만
- 불필요한 선택지 제거

### 흐름 중심

- "다음" 중심의 단일 흐름
- 명확한 진행 방향

### 크롬 최소화

- 상단 얇은 헤더만 (진행률 표시)
- 불필요한 메뉴/사이드바 제거

### 사용자 통제감 유지

- 세션 시작 전 미리보기 제공
- 중요한 단계에서 사용자 확인 요청
- "AI가 준비한 학습"이지만 사용자가 주도권을 가진다는 느낌 제공

## 단계별 UI

### 1. 신규 개념 학습

```
오늘 배울 내용

📖 [학습 콘텐츠 영역: 텍스트, 이미지, 예제 등]

스크롤 또는 다음 버튼으로 진행
```

### 2. 이해도 체크

```
잠깐, 확인해볼까요?

Q: [현재 학습한 개념에 대한 핵심 질문]

○ [보기 1]
● [보기 2 - 정답 선택]
○ [보기 3]

[확인]
```

### 3. 적용 활동

```
직접 해보기

아래의 빈칸을 채워 학습한 내용을 완성해보세요.

[제공된 문장이나 코드의 빈칸]

[____] 입력 필드

[정답 확인]
```

### 4. 세션 완료

```
🎉 오늘 학습 완료!

훌륭해요! 오늘도 열심히 학습하셨네요.

학습한 개념 3개가
Concept Library에 저장되었습니다.

💬 "이번 개념은 어려울 수 있지만,
    곧 익숙해질 거예요!"

[아카이브 보기]  [홈으로]
```

### 5. 개인화 설정 (선택)

```
⚙️ 학습 설정

난이도: [쉽게] ● [보통] [어렵게]
설명량: [간단히] [보통] ● [자세히]

💡 이 설정은 다음 세션부터 반영됩니다.
```

## 세션 종료 처리

### 정상 완료

1. Concept 자동 저장
2. 세션 요약 카드 생성
3. 홈으로 복귀 또는 아카이브 이동

### 중간 종료

1. 확인 다이얼로그: "학습을 중단하시겠습니까?"
2. 진행 상황 저장 여부 선택
3. 이어서 학습 가능

## 세션 복구 (Session Recovery) - 구현은 선택 사항

사용자가 학습 도중 네트워크가 끊기거나 실수로 페이지를 이탈하더라도, 다시 접속했을 때 중단된 지점부터 매끄럽게 학습을 이어서 할 수 있도록 지원합니다.

### 목표

- **중단 없는 경험**: 기술적 문제나 불가피한 상황으로 인한 학습 흐름 끊김 최소화
- **데이터 보존**: 사용자가 입력 중이던 답변이나 학습 진행 상황(Progress) 소실 방지

### 메커니즘

1. **자동 저장 (Auto-Save)**:
   - 학습 진행 단계(Step Index), 입력 중인 텍스트, 선택한 옵션 등을 **로컬 스토리지(Local Storage)** 와 **서버**에 주기적으로 동기화합니다.
   - 저장 주기는 사용자 입력 감지 시 또는 매 단계 전환 시 트리거됩니다.

2. **상태 감지 및 복구**:
   - 사용자가 `/session` 페이지에 재진입 시, 진행 중이던 세션 정보가 있는지 확인합니다.
   - 중단된 세션이 발견되면 이어서 학습을 할 수 있도록 합니다.

3. **네트워크 오류 대응**:
   - 오프라인 상태 감지 시, 화면에 "네트워크 연결이 불안정합니다. 연결 시 자동 재시도합니다." 알림을 표시하고, 입력 데이터는 로컬에 우선 저장합니다.
   - 연결 복구 시 자동으로 서버와 동기화하고 학습을 재개합니다.
