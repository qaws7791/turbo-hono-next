# 제품 원칙

## 핵심 UX 원칙

숏폼/이지 리스닝 트렌드처럼 '쉽게 소비되는 경험'을 지향하면 이탈이 줄어듭니다. 교수자 역할을 AI가 대체한다는 전제를 UX에 반영해야 합니다.

---

## 1. 단순성 (Simplicity)

### 배경

- 학습 앱에서 '선택'은 곧 피로와 미루기로 이어짐
- LMS 스타일의 메뉴/설정/편집 UI는 사용자의 인지 부담을 증가시킴

### 적용 원칙

- UI는 최소한의 선택만 제공
- LMS처럼 수십 개의 메뉴/페이지 없이, **핵심 화면**만 존재
- 학습은 별도 페이지를 늘리지 않고, 홈에서 바로 진입 가능한 **풀스크린 학습 세션 모드(View)**를 제공
- 사용자는 스스로 구조를 만들거나 설정할 필요 없음

---

## 2. 자동화 (Automation)

### 배경

- 교수자가 하던 일을 사용자가 대신하면 '자기주도'가 아니라 '자기관리 노동'이 됨
- AI 자동화가 일관되게 작동해야 사용자는 "시작 버튼만 누르면 된다"는 기대를 형성

### 적용 원칙

- 커리큘럼 생성, 난이도 조절, 복습 스케줄링을 자동 수행
- 학습 진행 상태는 AI가 지속적으로 추적
- 사용자는 **"학습 시작"**만 누르면 됨

---

## 3. 개인화 (Adaptive Personalization)

### 배경

- 개인화는 '추천'이 아니라 '경로/세션/복습의 자동 설계'일 때 체감 효과가 큼
- 세션은 작업기억 한계를 고려해 짧고 명확한 흐름으로 설계해야 함 (Cognitive Load)

### 적용 원칙

- 학습 경로는 **문서 분석 기반 Curriculum Structure(모듈 구조)**로 자동 생성
- 세션 구성은 **Cognitive Load Theory 기반**으로 20~40분 최적화
- 복습은 **Spaced Repetition 알고리즘**에 따라 자동 생성

---

## 4. 사용자 행동 관점 요약

### 배경

- 사용자의 "해야 할 일"을 1~2개로 줄여야 습관화(재방문)가 쉬움
- '진단'을 별도 과제로 분리하면 허들이 되므로, 학습 과정 내부로 흡수

### 핵심 행동

1. 사용자는 매일 **"세션 시작하기"**만 누르면 됨
2. 진단은 세션 안에서 자연스럽게 진행되며, 별도의 시험/과제 화면 없음
3. 학습이 끝나면 내용은 자동으로 **지식 아카이브(Concept Library)**에 저장
4. 방금 학습한 내용은 **최근 세션 요약 카드** 또는 **지식 아카이브**에서 확인 가능
5. 커리큘럼(모듈 구조)은 학습 방향을 보여주는 안내이며, 사용자가 관리/편집할 필요 없음

---

## 제품 정체성

> **"코스를 만드는 도구"가 아니라 "학습을 지속시키는 시스템"**

페이지 수와 선택지를 줄이는 것이 제품의 정체성(개인화 자기주도 학습 OS)을 강화합니다. 단순성(최소 행동)과 자동화(교수자 대체)를 끝까지 밀어야 제품 정체성이 흔들리지 않습니다.
