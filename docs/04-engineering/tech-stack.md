# 기술 스택 개요

## 개요

이 플랫폼은 **TypeScript** 중심의 풀스택 아키텍처로, 프론트엔드와 백엔드 모두 타입 안전성을 확보하고 빠른 개발 속도를 목표로 합니다.

---

## 목표

- **빠른 실험/반복**으로 '5페이지 + 풀스크린 세션 모드' 제품 구조 구현
- 프론트–백엔드–DB–AI를 하나의 타입 시스템(TypeScript)으로 연결
- **설계 신뢰도(재현성)와 개발 생산성** 동시 확보
- **포트폴리오 관점**: 2025년 프론트엔드 시장의 기본 요구인 TypeScript + AI 통합 역량 증명

---

## Frontend

| 기술                   | 용도                       |
| ---------------------- | -------------------------- |
| **React**              | UI 라이브러리              |
| **TypeScript**         | 타입 안전 개발             |
| **TanStack Router v7** | SPA/SSR Hybrid 라우팅      |
| **Vite**               | 빌드 도구                  |
| **TailwindCSS v4**     | 스타일링                   |
| **Shadcn UI**          | UI 컴포넌트 (Base-UI 기반) |
| **Tabler React**       | 아이콘                     |
| **TanStack Query**     | 서버 상태/캐싱 (선택)      |

### 의사결정 근거

- 본 기획의 핵심은 "세션 몰입(풀스크린 모드) + 즉시성(반복 학습)"
- 초기 MVP에서는 SSR/SEO보다 **빠른 반복과 UX 실험**이 더 중요
- 세션·아카이브·업로드 등 화면 전환이 잦은 앱에서 데이터 로딩 패턴을 안정화

### 포트폴리오 관점

- **React + TypeScript**: 2025년 필수 기술 스택
- **복잡한 상태 관리**: 풀스크린 세션, 멀티스텍 플로우 등 고급 UI 패턴 경험
- **현대적 도구 활용**: Vite, TailwindCSS v4, Shadcn UI 등 최신 기술 하단 이해

## Backend

| 기술            | 용도                   |
| --------------- | ---------------------- |
| **HonoJS**      | 경량 웹 프레임워크     |
| **TypeScript**  | 타입 안전 개발         |
| **Zod**         | 런타임 스키마 검증     |
| **zod-openapi** | OpenAPI 스펙 자동 생성 |

### 의사결정 근거

- 학습 플랫폼은 '업로드/분석/세션/아카이브/AI 대화' 등 API 종류가 빠르게 늘어남
- 계약(OpenAPI/Zod)을 중심으로 설계하면, 프론트와 백엔드가 서로 다른 해석으로 구현되는 위험을 낮춤
- 스키마는 문서화(신뢰)와 테스트(회귀 방지)에 직접 재사용

## Database

| 기술            | 용도                  |
| --------------- | --------------------- |
| **PostgreSQL**  | 주 데이터베이스       |
| **NeonDB**      | Serverless PostgreSQL |
| **Drizzle ORM** | 타입 안전 ORM         |
| **PGVector**    | 벡터 저장/검색        |

### 의사결정 근거

- 사용자/학습 계획/학습 세션/개념/진행률/AI 대화 로그 등 구조화된 데이터가 중심
- Drizzle은 타입 안정성을 유지하면서도 SQL 친화적
- 학습 데이터의 쿼리/리포팅/확장에 유리

## AI/ML

| 기술                  | 용도            |
| --------------------- | --------------- |
| **LangChain**         | RAG 파이프라인  |
| **OpenAI GPT-5-mini** | LLM 및 임베딩   |
| **unpdf**             | PDF 텍스트 추출 |

### 의사결정 근거

- LangChain은 RAG 파이프라인 구축에 유연함
- OpenAI는 임베딩과 생성 모두 지원
- PDF 처리를 위한 unpdf 라이브러리 사용

### 포트폴리오 관점

- **AI 통합 경험**: 프론트엔드에서 RAG, LLM, 벡터 검색 등 최신 AI 기술 적용
- **실무 적용 가능성**: 고객 대면 AI 기능 구현 경험
- **기술 트렌드**: 2025년 시장에서 요구되는 AI/LLM 활용 역량 증명
