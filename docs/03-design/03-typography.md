# 타이포그래피

## 의사결정 근거

- 학습은 '읽기/이해'가 핵심이므로 타이포 스케일과 행간이 UX 품질을 좌우
- 한국어 환경에서는 Pretendard 계열이 가독성과 시스템 일관성에 유리

---

## 폰트

### 기본 폰트

- **권장**: Pretendard
- **대안**: 시스템 폰트 스택

---

## 타입 스케일

| Class | 크기 (px) | rem    | 용도               |
| ----- | --------- | ------ | ------------------ |
| 2xl   | 26        | 1.625  | 페이지 제목        |
| xl    | 22        | 1.375  | 섹션 제목          |
| lg    | 18        | 1.125  | 소제목             |
| base  | 16        | 1.000  | 본문 텍스트 (Base) |
| sm    | 14        | 0.875  | 보조 텍스트        |
| xs    | 13        | 0.8125 | 메타데이터         |
| xxs   | 12        | 0.750  | 법적 고지, 태그    |

---

## 스타일 원칙

### 헤드라인

- 크고 간결하게
- 1줄 요약 중심
- 명확한 계층 구조

### 본문

- 행간 넉넉하게 (장문 피로 감소)
- 적절한 문단 간격
- 읽기 편한 글자 간격

---

## 가이드라인

### DO ✅

- 명확한 계층 구조 유지
- 충분한 행간 확보
- 가독성 우선

### DON'T ❌

- 과도한 폰트 크기 변화
- 좁은 행간
- 장식용 폰트 사용
