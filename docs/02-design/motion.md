# 모션 & 애니메이션 원칙

## 의사결정 근거

- 과한 모션은 학습 집중을 방해
- 대신 전환(다음 단계로 넘어감)에서 최소한의 피드백만 주면 흐름이 부드러워짐

---

## 기본 원칙

### 1. 미니멀 모션

- 최소한의 시각적 피드백만 제공
- 장식적 애니메이션 지양
- 기능적 목적이 있는 모션만 사용

### 2. 빠른 응답

- 사용자 인터랙션에 즉각적인 피드백
- 지연 없는 상태 전환

### 3. 자연스러운 흐름

- 급격한 변화 없이 부드러운 전환
- 연속성 있는 시각적 경험

---

## 타이밍

### 기본 전환

- **Duration**: 150–220ms
- **Easing**: ease-out 또는 ease-in-out

### 페이지 전환

- **Duration**: 200–300ms
- **Effect**: 페이드 또는 미세한 슬라이드

### 세션 단계 전환

- "다음" 클릭 시 부드러운 전환으로만 표현
- 과도한 효과 금지

### 성취/완료 순간

- **Duration**: 400–600ms
- **Effect**: 미묘한 스케일 업 또는 페이드 인 + 이모지 효과
- 예: 세션 완료 시 축하 메시지에 가벼운 카드 애니메이션
- 연속 학습일수 증가 시 불꽃 아이콘에 미묘한 펄스 효과

---

## 접근성

### 시스템 모션 설정 존중

OS(Windows, macOS 등) 설정에서 '애니메이션 효과 줄이기'를 선택한 사용자를 위해:

- 모든 애니메이션과 전환 효과를 즉각적인 전환으로 변경하거나 제거합니다.
- 학습 흐름이나 기능 작동에는 영향을 주지 않으면서 시각적 움직임만 최소화하여 전정 장애가 있는 사용자나 빠른 반응을 선호하는 사용자에게 최적의 경험을 제공합니다.

---

## 가이드라인

### DO ✅

- 기능적 목적의 모션
- 일관된 타이밍
- 접근성 설정 존중

### DON'T ❌

- 장식적 애니메이션
- 느린 전환 (>300ms)
- 과도한 움직임
- 자동 재생 애니메이션
