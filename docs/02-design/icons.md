# 아이콘 & 그래픽

## 의사결정 근거

- 아이콘은 정보 전달을 돕되, 텍스트를 대체하지 않음
- 스타일이 여러 개 섞이면 '학습 OS'의 정돈된 인상이 무너짐

---

## 아이콘 시스템

### 아이콘 라이브러리

- **권장**: Tabler Icons
- **스타일**: Outline, Filled 두 가지 사용

### 크기 규칙

| 용도                   | 크기           |
| ---------------------- | -------------- |
| 인라인 (텍스트와 함께) | 16px (1rem)    |
| 버튼 내부              | 20px (1.25rem) |
| 카드 아이콘            | 24px (1.5rem)  |
| 빈 상태 일러스트       | 48-64px        |

---

## 사용 원칙

### 1. 보조 역할

- 아이콘은 텍스트의 의미를 보조
- 아이콘만으로 정보 전달하지 않음
- 항상 라벨 또는 툴팁 제공

### 2. 일관성

- 전체 앱에서 동일한 아이콘 세트 사용
- 동일한 개념에는 동일한 아이콘

### 3. 간결함

- 단순하고 인식하기 쉬운 형태
- 과도한 세부 표현 지양

---

## 자주 사용되는 아이콘

| 개념      | 아이콘 예시   | 용도                   |
| --------- | ------------- | ---------------------- |
| 세션 시작 | `player-play` | CTA 버튼               |
| 설정      | `settings`    | 설정 메뉴              |
| 문서      | `file-text`   | 학습 자료              |
| 체크/완료 | `check`       | 완료 상태              |
| 잠금      | `lock`        | 잠긴 콘텐츠            |
| 검색      | `search`      | 검색 입력              |
| 추가      | `plus`        | 새 항목 생성           |
| 홈        | `home`        | 홈 네비게이션          |
| 경로      | `map`         | Curriculum (모듈 구조) |

---

## 일러스트레이션

### 기본 원칙

- 최소로 사용 (필요 시에만)
- 단색 또는 라인 기반
- 복잡한 일러스트 지양

### 사용 케이스

- **빈 상태**: 아직 데이터가 없을 때
- **온보딩**: 기능 소개
- **성공 상태**: 완료 축하

---

## 가이드라인

### DO ✅

- 일관된 아이콘 스타일
- 텍스트와 함께 사용
- 적절한 크기 유지

### DON'T ❌

- 여러 아이콘 스타일 혼용
- 아이콘만으로 정보 전달
- 과도한 장식적 사용
