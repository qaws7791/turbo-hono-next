# 레이아웃 & 여백 규칙

## 의사결정 근거

- 여백은 정보 밀도를 조절해 '학습 피로'를 줄이는 가장 강력한 도구
- 유동적인 레이아웃(Flex/Grid)은 다양한 디바이스와 콘텐츠에 유연하게 대응함

---

## 레이아웃 전략 (Flexible System)

### Hybrid Strategy: Grid + Flexbox

고정된 컬럼(Fixed Columns)이나 픽셀 단위의 경직된 그리드 시스템을 지양하고, **CSS Grid**와 **Flexbox**를 유기적으로 사용합니다.

- **CSS Grid (2차원)**: 페이지 전체 구조, 대시보드 그리드. `fr`, `minmax()`, `auto-fit` 등을 활용해 유동적으로 구성합니다.
- **Flexbox (1차원)**: 컴포넌트 내부 정렬, 네비게이션 바. 콘텐츠 크기에 따라 자연스럽게 늘어나거나 줄어듭니다.

### Spacing Scale

고정된 간격보다는 리듬감을 위한 8px 기반 스케일을 **가이드**로 사용합니다.

| 토큰      | 기준값 (rem) | 용도               |
| --------- | ------------ | ------------------ |
| `space-1` | 0.5rem       | 아이콘/텍스트 간격 |
| `space-2` | 1rem         | 카드 내부 패딩     |
| `space-3` | 1.5rem       | 섹션 간격          |
| `space-4` | 2rem         | 큰 섹션 구분       |

## 레이아웃 원칙

### 1. 정보 밀도 제어

- 카드/섹션 간 여백을 크게 유지
- 한 화면에 정보를 과도하게 채우지 않음

### 2. 단일 CTA 강조

- 한 화면에 한 가지 핵심 행동(CTA)만 강조
- 부차적인 액션은 시각적으로 약하게 처리

### 3. 콘텐츠 폭 제한

- 읽기 편안함을 위해 최대 폭 제한
- 데스크톱에서 1024px 권장

---

## 반응형 레이아웃

### 데스크톱 (≥960px)

- 사이드바 + 메인 콘텐츠 2컬럼 구조
- 중앙 정렬된 콘텐츠 영역

### 태블릿 (768px ~ 960px)

- 축소된 사이드바 또는 숨김
- 단일 컬럼 위주

### 모바일 (<768px)

- 하단 네비게이션 또는 햄버거 메뉴
- 완전한 단일 컬럼

---

## 가이드라인

### DO ✅

- Grid/Flexbox의 적절한 조화
- 유동적인 너비 (`fr`, `%`, `auto`) 사용
- 충분한 여백 확보
- 최대 폭 제한

### DON'T ❌

- 임의의 여백 값
- 빽빽한 정보 밀도
- 무제한 콘텐츠 폭
