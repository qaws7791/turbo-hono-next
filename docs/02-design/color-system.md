# 컬러 시스템

## 의사결정 근거

- 색이 많아질수록 사용자는 '선택'과 '의미'를 해석해야 하므로 피로가 증가
- 상태(진행/진단) 표현을 위해서는 최소한의 시맨틱 컬러가 필요하지만, 채도를 낮춰 시각적 소음을 줄임

---

## 기본 팔레트

### Base Colors

- **기본**: Warm-neutral(오프화이트/미색) + 그레이 스케일 (Tailwind Stone 권장)
- **Accent**: 강조색 1개만 사용
  - Primary CTA, 핵심 진행률, 중요한 하이라이트에만 적용

### 시맨틱 컬러 (채도 낮춤)

| 상태           | 용도      | 적용 범위              |
| -------------- | --------- | ---------------------- |
| Success (초록) | 성공/충분 | 모듈/세션 완료 상태    |
| Warning (노랑) | 주의      | 모듈/세션 진행 중 상태 |
| Danger (빨강)  | 부족      | 모듈/세션 미완료 상태  |

> ⚠️ 시맨틱 컬러는 모듈/세션 상태에만 제한적으로 사용

---

## 디자인 토큰

### 배경 및 서피스

- **기본 배경**: 메인 화면의 배경색입니다.
- **컨테이너 배경**: 카드나 섹션 등의 배경색입니다.
- **중첩 컨테이너 배경**: 배경 위에 겹쳐진 요소의 구분감이 필요할 때 사용합니다.

### 텍스트 및 경계선

- **기본 텍스트**: 주요 본문에 사용되는 텍스트 색상입니다.
- **보조 텍스트**: 부가 설명이나 덜 중요한 정보에 사용되는 흐릿한 텍스트 색상입니다.
- **경계선**: 요소 간의 구분선에 사용됩니다.

### 강조색 (Accent)

- **주요 강조**: 사용자에게 가장 중요한 행동(버튼 등)을 유도할 때 사용합니다.
- **보조 강조**: 마우스 커서를 올렸을 때(호버) 등 부수적인 강조가 필요할 때 사용합니다.

### 상태 안내색

- **성공/완료**: 학습이나 동작이 완료되었음을 알리는 색상입니다.
- **주의/진행**: 작업이 진행 중이거나 주의가 필요할 때 사용합니다.
- **오류/부족**: 문제가 발생했거나 보충이 필요할 때 사용하며, 사용 빈도를 엄격히 제한합니다.

---

## 사용 가이드라인

### 권장 사항 (DO) ✅

- 강조색은 '시작하기'와 같은 핵심 행동 버튼에만 집중해서 사용하세요.
- 상태 안내색은 학습의 진행 정도를 보여줄 때만 제한적으로 사용하세요.
- 눈의 피로를 줄이기 위해 전반적으로 은은한 톤을 유지하세요.

### 지양 사항 (DON'T) ❌

- 한 화면에서 너무 많은 강조색을 동시에 사용하지 마세요.
- 너무 원색에 가까운 진한 상태 안내색은 사용자가 불안감을 느낄 수 있으므로 피하세요.
- 단순히 장식을 목적으로 색상을 남용하지 마세요.

---

## 상태별 색상 적용 예시

학습 모듈이나 세션의 상태에 따라 다음과 같이 색상을 적용하여 직관적으로 정보를 전달합니다.

- **완료됨**: 긍정적인 느낌을 주는 초록색 톤의 배경을 사용하여 성취감을 줍니다.
- **진행 중**: 주의를 끌 수 있는 노란색 톤의 배경을 사용하여 현재 위치를 나타냅니다.
- **잠김/미완료**: 아직 수행되지 않았거나 부족한 부분은 빨간색 톤으로 표시하여 보충을 유도합니다.
